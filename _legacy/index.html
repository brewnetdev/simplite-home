Philosophy

Tools should disappear into the work.
Not become it.

도구를 배우는 데 시간을 쓰지 않아야 합니다. Simplite는 당신이 원래 하려던 일 — 생각하고, 쓰고, 결정하고, 배포하는 일 — 에만 집중하도록 만들어졌습니다. 단순함은 제약이 아니라, 가장 어려운 설계의 결과입니다.

01 / SIMPLICITY

따로 배울 필요 없이

처음 여는 사람도 5분 안에 자기 흐름을 찾을 수 있어야 합니다. Simplite의 모든 제품은 1페이지 매뉴얼 원칙 위에서 설계됩니다.

02 / OPENNESS

열려 있는 기본값

코어 기능은 Apache 2.0으로 공개되어, 직접 수정·자가호스팅·상업적 이용까지 자유롭습니다. 잠금이 아니라 신뢰로 시작합니다.

03 / AI-NATIVE

MCP와 에이전트 위에서

모든 데이터와 업무 단위는 MCP 네이티브로 구조화되어, 당신의 에이전트와 업무 흐름에 자연스럽게 연결됩니다.

FAQ

자주 묻는 질문.

Free(오픈소스) 버전은 정말 영구 무료인가요? +
네. Tika와 MarkFlow의 코어는 Apache 2.0으로 공개되어 있으며, 개인·상업적 이용·수정·재배포 모두 자유롭습니다. 자가호스팅 시 별도 비용이 없으며, 커뮤니티 지원은 GitHub Issues에서 이뤄집니다.
Team / Enterprise 플랜은 언제 출시되나요? +
현재 고도화 개발 중이며, 2026년 상반기 베타 출시를 목표로 진행하고 있습니다. 문의 페이지에서 대기 리스트에 등록하시면 우선 순위로 안내드립니다.
Tika와 MarkFlow는 따로 써도 되나요? +
각각 독립 제품으로 설계되어 단독 사용이 가능합니다. 다만 MCP 네이티브 구조 덕분에, 둘을 함께 쓰면 MarkFlow의 문서와 Tika의 티켓이 자연스럽게 연결되어 업무 컨텍스트를 유지할 수 있습니다.
온프레미스 LLM은 어떤 모델을 지원하나요? +
Ollama, vLLM 기반의 오픈 모델(Llama 3, Qwen, DeepSeek 등)을 기본 지원합니다. OpenAI 호환 API 엔드포인트라면 무엇이든 연결 가능하며, 사내 인프라 구축도 도와드립니다.
기존 업무 도구(Jira, Notion 등)에서 마이그레이션이 가능한가요? +
Tika는 Jira CSV Import를, MarkFlow는 Notion·Confluence·docx·pdf의 MD 일괄 변환을 지원합니다. Enterprise 플랜은 커스텀 마이그레이션 지원을 포함합니다.
MCP 네이티브는 정확히 무엇을 의미하나요? +
모든 티켓·문서·워크스페이스 오브젝트가 Model Context Protocol 스펙으로 노출된다는 의미입니다. Claude Desktop, Cursor, 자사 AI 에이전트에서 Simplite 데이터에 직접 접근·수정할 수 있습니다.
